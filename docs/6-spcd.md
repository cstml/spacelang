# spcd ‚Äî spacelang package manager

Dependency fetcher inspired by Go modules. Reads a manifest (`deps.sp`), clones dependencies into `spcd_lib/`, writes a lockfile (`lock.sp`).

## What it does

`spcd` manages spacelang dependencies. You declare what you need in `deps.sp`, run `spcd fetch`, and it clones them into `spcd_lib/`. The runtime resolver (`:require` in `spci` and `spcc`) finds them there automatically via `lock.sp`.

```
deps.sp  ‚îÄ‚îÄ‚ñ∫  spcd fetch  ‚îÄ‚îÄ‚ñ∫  spcd_lib/<dep>/  +  lock.sp
```

## Verbs

| Verb | Effect |
|---|---|
| `spcd add URL[@ref]` | Append a dependency line to `deps.sp`, then fetch |
| `spcd fetch` | Read `deps.sp`, clone deps, write `lock.sp` |
| `spcd update` | Re-resolve all deps, rewrite `lock.sp` |
| `spcd install URL` | Fetch + spcc-compile + install binary to `$PATH` |
| `spcd list` | Print `lock.sp` contents |
| `spcd clean` | `rm -rf spcd_lib/` |

## deps.sp format

The manifest file is actually a spacelang file itself. 

```
{ Declare a dependency }
"github.com/cstml/spacelang" "packaging" deps/branch

{ Override a URL to a local path (like Go's replace directive) }
"../spacelang" "github.com/cstml/spacelang" deps/override
```

### Dep kinds

| Kind | Example | Meaning |
|---|---|---|
| `deps/head` | `"url" deps/head` | Track default branch HEAD |
| `deps/branch` | `"url" "branch" deps/branch` | Track a branch |
| `deps/tag` | `"url" "v1.0" deps/tag` | Pin to a tag |
| `deps/sha` | `"url" "abc123..." deps/sha` | Pin to a specific commit |
| `deps/override` | `"local" "url" deps/override` | Replace URL with local path |

### Ref inference for `spcd add`

When you run `spcd add URL@ref`, the ref is heuristically classified:

- 40 hex chars ‚Üí `deps/sha`
- Starts with `v` ‚Üí `deps/tag`
- Anything else ‚Üí `deps/branch`
- No ref ‚Üí `deps/head`

```
spcd add github.com/cstml/spacelang@packaging    ‚Üí deps/branch
spcd add github.com/cstml/spacelang@v0.0.1       ‚Üí deps/tag
spcd add github.com/cstml/spacelang              ‚Üí deps/head
```

## lock.sp

Auto-generated by `spcd fetch`. Pins every dependency (direct and transitive) to an exact SHA. Example:

```
{ generated by spcd -- do not edit by hand }
"github.com/cstml/spacelang" "cd7c4b2b02ed7d480aa56482bca2b198f332ae40" deps/sha
```

The C resolver in `spci` and `spcc` parses `lock.sp` to find deps in `spcd_lib/` ‚Äî no manual `deps/override` needed.

## Transitive dependencies

After cloning a dep, `spcd` checks for a `deps.sp` inside the cloned repo. If found, it evaluates it to discover sub-dependencies. All end up pinned in `lock.sp`.

## Install

`spcd install URL` fetches a repo, compiles its `main.sp` with `spcc`, and installs the binary to `$SPACELANG_BIN` (default: `$HOME/.local/bin`).

```bash
spcd install github.com/cstml/some-tool
```
