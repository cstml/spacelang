# spcc — spacelang compiler

Compiles `.sp` source into standalone native executables. AOT compilation via C: your spacelang becomes a small C file, then `cc` compiles and links it against `libspci.a`.

## What it does

`spcc` reads a `.sp` source file (with `:require` preprocessing), emits a generated C file, and shells out to a C compiler. The output binary is fully self-contained — no runtime dependency beyond libc.

```
hello.sp  ──►  spcc  ──►  generated C  ──►  cc + libspci.a  ──►  ./hello
```

The generated C is tiny (~1 KB): a `#include "spci.h"`, the source as a byte array, and a `main()` that calls `feed(SP_SOURCE)`.

## CLI

```
spcc input.sp -o output           compile to ./output
spcc --emit-c input.sp            print generated C to stdout
spcc --keep-c input.sp -o output  also write output.c next to binary
spcc --cc gcc input.sp -o output  override $CC (default: cc)
spcc --debug input.sp -o output   compile with -g -O0, implies --keep-c
spcc --root DIR input.sp ...      override runtime-source lookup
```

## :require preprocessing

`spcc` statically resolves `:require` at compile time. When it sees `"path" :require`, it opens that file, inlines its contents, and recursively processes any `:require` directives within it. Resolution order:

1. **deps/override** from `deps.sp` — explicit URL→local-path mappings
2. **lock.sp** (auto-generated by `spcd fetch`) — implicit `spcd_lib/<basename>/` mappings
3. **Sibling paths** relative to the requiring file

The output binary contains everything inlined — no runtime file reads.

## Runtime lookup

`spcc` needs `spci.h` and `libspci.a` to compile. It finds them, in order:

1. `$SPACELANG_ROOT` (if set)
2. The directory containing the `spcc` binary itself

The first location containing both files wins. Installed via `make install` (goes to `~/.local/` by default).

## No runtime dependencies

The output binary is fully self-contained. `libspci.a` is statically linked at compile time — the binary has no dependency on spacelang at runtime. No interpreter, no library, no headers. Just the binary and libc. Copy it to any Linux machine and run it.

```bash
spcc hello.sp -o hello
scp hello other-machine:/tmp/
ssh other-machine /tmp/hello
# Hello, spacelang!
```

## Mesh support

Compiled binaries speak the same mesh protocol as `spci`. Identity comes from:

- `SPACELANG_NAME` / `SPACELANG_BUS` environment variables
- `--name` / `--bus` flags on the compiled binary
- `--serve` keeps the process alive after `feed()` returns

```bash
spcc worker.sp -o worker
SPACELANG_NAME=W1 SPACELANG_BUS=/tmp/spacelang ./worker --serve
```
